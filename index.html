데이터, 시각화를 만나기 위한 첫걸음: 데이터 정제 및 가공

시작하며: 왜 데이터 정제가 중요한가?

"요리할 때 재료를 다듬는 것처럼, 데이터를 분석하고 시각화하기 전에는 반드시 '데이터 다듬기' 과정이 필요합니다. 우리는 이것을 데이터 정제(Data Cleaning) 또는 데이터 전처리(Data Preprocessing)라고 부릅니다."

우리가 사용할 데이터는 넷플릭스의 콘텐츠 정보를 담고 있는 CSV 파일입니다. 각 행(row)은 개별 콘텐츠를, 각 열(column)은 콘텐츠의 속성을 나타냅니다. 이런 구조화된 데이터를 정형 데이터라고 부릅니다.

[1부] 잘 차려진 밥상: 정제된 데이터와 시각화

대표적인 시각화 차트들을 살펴보며, 각 차트를 그리기 위해 데이터가 어떤 구조를 가져야 하는지 알아보겠습니다.

막대형 차트 (Bar Chart): 카테고리별 크기 비교

"국가별 넷플릭스 콘텐츠는 얼마나 될까?" 국가별 콘텐츠 수를 비교하여 상위 10개국을 막대 차트로 나타냅니다.

선형 차트 (Line Chart): 시간의 흐름에 따른 변화

"연도별 넷플릭스 콘텐츠 출시량 변화는?" 2000년 이후 연도별 콘텐츠 출시량의 변화 추이를 선형 차트로 보여줍니다.

원형 차트 (Pie Chart): 전체에 대한 각 부분의 비율

"넷플릭스에는 영화가 많을까, TV 쇼가 많을까?" 전체 콘텐츠에서 Movie와 TV Show가 차지하는 비율을 원형 차트로 확인합니다.

히스토그램 (Histogram): 데이터의 분포 확인

"넷플릭스 영화들의 상영 시간(duration) 분포는 어떨까?" 영화들의 상영 시간이 어떤 구간에 가장 많이 분포하는지 히스토그램으로 분석합니다.

[2부] 다른 그릇에 담기: JSON 데이터와 그 외

데이터는 다양한 형식으로 존재합니다. 테이블 형태의 데이터에 적합한 CSV와 달리, 계층 구조를 표현하는 데 용이한 JSON 형식에 대해 알아봅니다. 아래 트리맵 차트는 콘텐츠 타입(Movie/TV Show)이라는 상위 계층과 그에 속한 등급(Rating)이라는 하위 계층의 분포를 시각적으로 보여주는 좋은 예시입니다.

트리맵 (Treemap): 계층 구조 데이터 시각화

"콘텐츠 타입별, 그리고 그 안에서의 등급별 분포는?"


[
  {
    "type": "Movie",
    "ratings": [
      { "rating": "PG-13", "count": 490 },
      { "rating": "R", "count": 799 }
    ]
  },
  {
    "type": "TV Show",
    "ratings": [
      { "rating": "TV-MA", "count": 1145 },
      { "rating": "TV-14", "count": 733 }
    ]
  }
]

JSON(JavaScript Object Notation)은 웹에서 데이터를 주고받을 때 표준처럼 사용되며, 트리맵, 네트워크 그래프 등 중첩된 구조를 시각화할 때 매우 편리합니다.

[3부] 날것의 재료 다듬기: Raw 데이터 정제

실제 세상의 데이터는 훨씬 복잡하고 지저분합니다. 웹에서 데이터를 수집하는 크롤링의 현실과, 문장으로 이루어진 텍스트 데이터를 정제하는 방법에 대해 알아봅니다.

Raw 데이터 수집의 현실

웹사이트의 정책(robots.txt), 서비스 약관, 기술적 장벽 등 때문에 데이터 수집은 기술뿐만 아니라 정책과 윤리를 함께 고려해야 하는 신중한 작업입니다.

문장형 텍스트 데이터 정제

텍스트 데이터를 분석하려면 의미를 가진 가장 작은 단위인 '형태소'로 문장을 분리해야 합니다. 이 과정은 언어별로 난이도가 다릅니다.

  • 영어: 띄어쓰기로 단어 구분이 가능해 매우 쉽습니다.
  • 한국어: 조사, 어미가 붙어있어 '형태소 분석기'가 필요합니다.
  • 중국어/일본어: 띄어쓰기가 거의 없어 단어 분리가 매우 어렵습니다.

정제된 텍스트 데이터는 '워드 클라우드' 같은 시각화로 표현할 수 있습니다.
